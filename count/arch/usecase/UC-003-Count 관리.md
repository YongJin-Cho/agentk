# UC-003-Count 관리

## 개요

### Use Case ID
UC-003

### 제목
Count 관리

### 설명
관리자가 웹 UI를 통해 Count 데이터를 생성, 수정, 삭제하는 통합 관리 기능을 수행합니다.

## 액터

### Primary Actor
관리자
- **역할**: Count 데이터를 직접 관리하고 모니터링하는 운영 담당자
- **설명**: 웹 UI를 통해 Count 데이터의 생성, 수정, 삭제를 수행하는 시스템 관리자

### Secondary Actor
없음

## 사전조건

- 관리자가 시스템에 로그인되어 있어야 함
- 관리자가 Count 관리 권한을 보유하고 있어야 함
- 시스템이 정상적으로 동작 중이어야 함

## 사후조건

- Count 데이터가 생성, 수정, 또는 삭제됨
- 변경된 Count 데이터가 시스템에 반영됨
- 관리자가 변경 사항을 확인할 수 있음

## 주요 시나리오

### 시나리오 1: Count 생성

1. 관리자가 웹 UI에서 Count 생성 화면으로 이동한다.
2. 관리자가 Count 정보를 입력한다. (Count ID, 초기 값 등)
3. 관리자가 생성 요청을 제출한다.
4. 시스템은 요청을 검증한다. (Count ID 중복 여부, 값 유효성 등)
5. 시스템은 새로운 Count 데이터를 생성한다.
6. 시스템은 Count 데이터를 저장한다.
7. 시스템은 관리자에게 생성 완료 메시지를 표시한다.

### 시나리오 2: Count 수정

1. 관리자가 웹 UI에서 Count 목록을 조회한다.
2. 관리자가 수정할 Count를 선택한다.
3. 관리자가 Count 정보를 수정한다. (Count 값 등)
4. 관리자가 수정 요청을 제출한다.
5. 시스템은 요청을 검증한다. (값 유효성 등)
6. 시스템은 Count 데이터를 업데이트한다.
7. 시스템은 업데이트된 Count 데이터를 저장한다.
8. 시스템은 관리자에게 수정 완료 메시지를 표시한다.

### 시나리오 3: Count 삭제

1. 관리자가 웹 UI에서 Count 목록을 조회한다.
2. 관리자가 삭제할 Count를 선택한다.
3. 관리자가 삭제 요청을 제출한다.
4. 시스템은 삭제 확인을 요청한다.
5. 관리자가 삭제를 확인한다.
6. 시스템은 Count 데이터를 삭제한다.
7. 시스템은 관리자에게 삭제 완료 메시지를 표시한다.

## 대안 시나리오

### 4a. Count ID 중복
생성 시 Count ID가 이미 존재하는 경우

4a.1. 시스템은 Count ID 중복 오류를 표시한다.
4a.2. 시스템은 관리자에게 다른 Count ID를 사용하도록 안내한다.
4a.3. 관리자가 다른 Count ID를 입력하고 다시 제출한다.

## 예외 시나리오

### E1. 인증 실패
관리자가 로그인되어 있지 않거나 세션이 만료된 경우

E1.1. 시스템은 로그인 화면으로 리다이렉트한다.
E1.2. 시스템은 요청을 처리하지 않고 종료한다.

### E2. 권한 부족
관리자가 Count 관리 권한을 보유하지 않은 경우

E2.1. 시스템은 권한 부족 오류를 표시한다.
E2.2. 시스템은 요청을 처리하지 않고 종료한다.

### E3. 요청 형식 오류
입력된 데이터의 형식이 올바르지 않거나 필수 필드가 누락된 경우

E3.1. 시스템은 입력 오류를 표시한다.
E3.2. 시스템은 관리자에게 올바른 형식으로 입력하도록 안내한다.

### E4. Count 값 유효성 검증 실패
Count 값이 허용 범위를 벗어나거나 유효하지 않은 경우

E4.1. 시스템은 유효성 검증 오류를 표시한다.
E4.2. 시스템은 관리자에게 유효한 값을 입력하도록 안내한다.

### E5. Count 데이터 없음
수정 또는 삭제 시 Count 데이터가 존재하지 않는 경우

E5.1. 시스템은 Count를 찾을 수 없다는 오류를 표시한다.
E5.2. 시스템은 요청을 처리하지 않고 종료한다.

### E6. 시스템 오류
시스템 내부 오류가 발생한 경우

E6.1. 시스템은 내부 오류를 기록한다.
E6.2. 시스템은 관리자에게 시스템 오류 메시지를 표시한다.
