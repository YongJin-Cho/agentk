# QS-017-대시보드-사용-편의성

## 개요

### Quality Scenario ID
QS-017

### 제목
대시보드 사용 편의성

### 설명
관리자가 대시보드를 사용하여 Count 데이터를 모니터링하는 편의성

### 품질 속성
사용성 (사용자 편의성)

## 환경

### 시스템 상태
- 시스템이 정상 가동 중
- Count 통합 관리 시스템이 Kubernetes 환경에서 MSA로 배포되어 있음
- 대시보드가 정상 동작 중

### 초기 조건
- 관리자가 대시보드에 접근 가능한 상태
- DashboardUI가 정상 동작 중
- 관리자가 대시보드 사용 방법을 알고 있음 (기본 사용법 숙지)
- 대시보드에 표시할 Count 데이터가 존재함

### 부하 조건
- 시스템이 정상 부하 상태에서 동작 중
- 대시보드에 다수의 Count 데이터가 표시될 수 있음

### 관련 컴포넌트
- DashboardUI (Boundary)
- DashboardManager (Control)

## 동작

1. 관리자가 대시보드를 열어서 Count 데이터 모니터링 시작
2. 관리자가 원하는 정보를 찾기 위해 대시보드 탐색
   - 예: 특정 Count 값 확인
   - 예: Count 변화 추이 확인
   - 예: 통계 정보 확인
3. 관리자가 필요한 정보를 찾거나 원하는 작업을 완료
   - 예: 원하는 Count 값을 찾음
   - 예: Count 변화 추이를 확인함
   - 예: 통계 정보를 확인함

## 측정

### 측정 항목
대시보드 사용 편의성

### 측정 공식
```
사용 편의성 = f(T_find, N_clicks)
```

여기서:
- `T_find`: 관리자가 원하는 정보를 찾는 데 걸리는 시간 (초)
- `N_clicks`: 관리자가 원하는 정보를 찾기 위해 수행한 클릭 수

### 측정 방법
- 관리자가 대시보드에서 특정 정보를 찾는 작업을 수행
- 정보를 찾는 데 걸리는 시간을 측정
- 정보를 찾기 위해 수행한 클릭 수를 측정
- 측정은 정보의 위치, 대시보드 구성, 사용자 경험에 따라 달라질 수 있음을 고려
