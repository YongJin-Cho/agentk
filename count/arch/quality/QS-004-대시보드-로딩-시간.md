# QS-004-대시보드-로딩-시간

## 개요

### Quality Scenario ID
QS-004

### 제목
대시보드 로딩 시간

### 설명
관리자가 대시보드를 열었을 때 초기 데이터를 로딩하는 시간

### 품질 속성
성능 (응답 시간)

## 환경

### 시스템 상태
- 시스템이 정상 가동 중
- Count 통합 관리 시스템이 Kubernetes 환경에서 MSA로 배포되어 있음
- 데이터베이스가 정상 동작 중

### 초기 조건
- 관리자가 Count 통합 관리 시스템에 접근 가능한 상태
- DashboardUI가 정상 동작 중
- DashboardConfigDB, CountInfoDB, CountValueDB가 정상 동작 중
- 네트워크 연결이 정상 상태
- 인증 및 권한 검증이 완료된 상태
- 대시보드 구성 정보가 DashboardConfigDB에 저장되어 있음

### 부하 조건
- 시스템이 정상 부하 상태에서 동작 중
- 다른 서비스의 요청이 동시에 처리되고 있을 수 있음
- 데이터베이스에 다수의 Count 데이터가 존재할 수 있음
- 대시보드에 표시할 Count 목록의 수에 따라 로딩 시간이 달라질 수 있음

### 관련 컴포넌트
- DashboardUI (Boundary)
- DashboardManager (Control)
- CountReader (Control)
- DashboardConfigDB (Entity)
- CountInfoDB (Entity)
- CountValueDB (Entity)

## 동작

1. 관리자가 대시보드 페이지 요청
2. DashboardUI가 인증 및 권한 확인 수행
3. DashboardUI가 DashboardManager로 대시보드 데이터 조회 요청 전달
4. DashboardManager가 DashboardConfigDB에서 대시보드 구성 조회
   - 표시할 Count 목록
   - 표시할 분석 데이터 목록
   - 요소별 표시 옵션
5. DashboardManager가 CountReader를 통해 Count 목록 및 현재 값 조회 요청
6. CountReader가 CountInfoDB에서 Count 목록 조회
7. CountReader가 CountValueDB에서 각 Count의 현재 값 조회
8. DashboardManager가 Count 데이터 통계 정보 계산
   - 총 Count 수
   - 평균 값
   - 최근 업데이트 시간 등
9. DashboardManager가 최근 기간의 Count 값 목록 조회 요청 (변화 추이용)
10. CountReader가 CountValueDB에서 최근 기간의 Count 값 목록 조회
11. DashboardManager가 대시보드 데이터 조합
    - Count 모니터링 정보
    - 통계 정보
    - 변화 추이 데이터
12. DashboardManager가 대시보드 데이터를 DashboardUI로 반환
13. DashboardUI가 대시보드 화면을 구성하여 관리자에게 표시

## 측정

### 측정 항목
대시보드 로딩 시간 (Loading Time)

### 측정 공식
```
로딩 시간 = T_display - T_request
```

여기서:
- `T_request`: 관리자가 대시보드 페이지 요청을 수행한 시각
- `T_display`: 관리자가 대시보드 화면에 데이터가 표시된 것을 확인한 시각

### 측정 방법
- 대시보드 페이지 요청 시각과 화면 표시 완료 시각을 기록
- 로딩 시간은 밀리초(ms) 단위로 측정
- 측정은 대시보드 구성 요소 수, 표시할 Count 수에 따라 달라질 수 있음을 고려
