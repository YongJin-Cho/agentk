# QS-019-입력-오류-복원-용이성

## 개요

### Quality Scenario ID
QS-019

### 제목
입력 오류 복원 용이성

### 설명
관리자의 잘못된 Count 관리 요청에 대한 복원 용이성

### 품질 속성
사용성 (오류 복원)

## 환경

### 시스템 상태
- 시스템이 정상 가동 중
- Count 통합 관리 시스템이 Kubernetes 환경에서 MSA로 배포되어 있음
- Count 관리 기능이 정상 동작 중

### 초기 조건
- 관리자가 Count 관리 화면에 접근 가능한 상태
- CountManagementUI가 정상 동작 중
- 관리자가 Count 관리 작업을 수행 중
- 잘못된 Count 관리 요청이 발생함
   - 예: 잘못된 Count 값으로 수정
   - 예: 잘못된 Count 삭제
   - 예: 잘못된 Count 생성

### 부하 조건
- 시스템이 정상 부하 상태에서 동작 중
- 오류 복원 작업이 필요함

### 관련 컴포넌트
- CountManagementUI (Boundary)
- CountManager (Control)
- CountInfoDB (Entity)
- CountValueDB (Entity)

## 동작

1. 관리자가 잘못된 Count 관리 요청 수행
   - 예: 잘못된 Count 값으로 수정 요청
   - 예: 잘못된 Count 삭제 요청
2. 시스템이 잘못된 요청을 처리하여 Count 데이터가 변경됨
3. 관리자가 오류를 인지함
4. 관리자가 오류 복원 작업 수행
   - 예: 이전 Count 값으로 복원
   - 예: 삭제된 Count 복구
   - 예: 잘못 생성된 Count 삭제
5. 관리자가 오류 복원 작업 완료

## 측정

### 측정 항목
입력 오류 복원 용이성

### 측정 공식
```
복원 용이성 = T_recovery
```

여기서:
- `T_recovery`: Count 관리 오류를 복원하는 데까지 걸리는 시간 (초)

### 측정 방법
- 관리자가 잘못된 Count 관리 요청을 수행한 후 오류를 인지
- 오류 복원 작업을 시작한 시각을 기록
- 오류 복원 작업이 완료된 시각을 기록
- 복원 시간은 복원 작업 시작 시각부터 완료 시각까지의 시간으로 측정
- 측정은 오류 유형, 복원 방법의 복잡도에 따라 달라질 수 있음을 고려
