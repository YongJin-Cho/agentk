# QS-018-분석-기능-사용-편의성

## 개요

### Quality Scenario ID
QS-018

### 제목
분석 기능 사용 편의성

### 설명
관리자가 분석 기능을 사용하여 원하는 분석을 수행하는 편의성

### 품질 속성
사용성 (사용자 편의성)

## 환경

### 시스템 상태
- 시스템이 정상 가동 중
- Count 통합 관리 시스템이 Kubernetes 환경에서 MSA로 배포되어 있음
- 분석 기능이 정상 동작 중

### 초기 조건
- 관리자가 분석 화면에 접근 가능한 상태
- CountAnalysisUI가 정상 동작 중
- 관리자가 분석 기능 사용 방법을 알고 있음 (기본 사용법 숙지)
- 분석 대상 Count 데이터가 존재함

### 부하 조건
- 시스템이 정상 부하 상태에서 동작 중
- 분석 대상 데이터량에 따라 처리 시간이 달라질 수 있음

### 관련 컴포넌트
- CountAnalysisUI (Boundary)
- CountAnalyzer (Control)

## 동작

1. 관리자가 분석 화면에 접근
2. 관리자가 분석 요청을 수행하기 위해 입력 작업 수행
   - 분석 유형 선택 (트렌드 분석, 비교 분석, 예측 분석)
   - 분석 대상 Count ID 선택
   - 분석 기간 설정
   - 기타 분석 옵션 설정
3. 관리자가 분석 실행 요청
4. 관리자가 분석 결과를 확인

## 측정

### 측정 항목
분석 기능 사용 편의성

### 측정 공식
```
사용 편의성 = f(T_complete, N_steps)
```

여기서:
- `T_complete`: 관리자가 분석 요청을 완료하는 데 걸리는 시간 (초)
- `N_steps`: 관리자가 분석 요청을 완료하기 위해 수행한 입력 단계 수

### 측정 방법
- 관리자가 분석 기능을 사용하여 특정 분석을 수행하는 작업을 수행
- 분석 요청을 완료하는 데 걸리는 시간을 측정
- 분석 요청을 완료하기 위해 수행한 입력 단계 수를 측정
- 측정은 분석 유형, 분석 옵션의 복잡도, UI 설계에 따라 달라질 수 있음을 고려
